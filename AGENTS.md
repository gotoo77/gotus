# Instructions de contribution pour les agents

## Changelog

- Mettre à jour la section `[À venir]` de `CHANGELOG.md` dans la même modification que toute évolution notable.
- Documenter l’impact fonctionnel, utilisateur, accessibilité, sécurité, données ou déploiement ; ne pas recopier les détails internes du diff.
- Ne pas ajouter d’entrée pour un simple formatage, renommage interne, test isolé ou correction typographique mineure.
- Utiliser uniquement les catégories nécessaires parmi `Ajouté`, `Modifié`, `Corrigé`, `Accessibilité`, `Sécurité`, `Déprécié` et `Supprimé`.
- Lors d’une release, garder `package.json`, `assets/data/build-info.json`, `CHANGELOG.md` et le tag Git cohérents.
- Suivre `docs/RELEASING.md` pour préparer une release.
